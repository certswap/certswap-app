Імпорт: MetaMask (Web3) → Backend

Отримайте токени з MetaMask. Користувач Web3 режиму створює gift — вставте gift hash нижче.

Як це працює

→

Backend → Web3: Backend генерує gift hash через системний гаманець. MetaMask користувач отримує його через claimGift(giftHash).

←

Web3 → Backend: MetaMask користувач створює gift в Web3 режимі. Backend отримує gift hash і виконує claimGift через системний гаманець — баланс зараховується на Backend рахунок.

Опис модулів

M1

Web3 / DeFi / MetaMask — некастодіальний режим. Користувач взаємодіє напряму зі смарт-контрактом через MetaMask. P2P-обмін, повний контроль ключів. Потрібен POL на Polygon Amoy.

M2

Backend / B2B лояльність / Нотаріус — кастодіальний режим. Блокчейн невидимий для користувача. Email або телефон замість гаманця. eIDAS-сумісний нотаріальний запис. Антиконтрафакт для FMCG/HoReCa.

M⇄

Міст M1↔M2 — CertSwap як нотаріальний реєстр обміну між режимами. Конвертація між DeFi-активами та B2B-балансами. eIDAS-сумісний bridge-запис.

Доступ до сайту (SITE_PASSWORD)

Basic Auth для кінцевих користувачів — захищає весь сайт паролем крім /api/ та /admin.

Як змінити на Render

  1. Відкрий Render Dashboard → обери свій сервіс certswap
  2. Зліва → Environment
  3. Знайди або додай змінну SITE_PASSWORD
  4. Щоб увімкнути — встанови значення пароля (будь-який рядок)
  5. Щоб вимкнути — видали змінну або залиш порожній рядок
  6. Натисни Save Changes → Render автоматично перезапустить сервер
Зміни набувають чинності після перезапуску (~30 сек). Адмін-панель (/admin) завжди доступна незалежно від SITE_PASSWORD.

Змінні середовища

  1. ADMIN_PASSWORD — пароль цієї адмін-панелі (Basic Auth + X-Admin-Token).
  2. SITE_PASSWORD — Basic Auth для кінцевих користувачів (необов'язково).
  3. SESSION_SECRET — секрет для підпису сесійних кук.
  4. CONTRACT_ADDRESS — адреса смарт-контракту Polygon Amoy.
  5. PRIVATE_KEY — ключ системного гаманця (тільки для Backend режиму).
Зміни env-змінних потребують перезапуску сервера. Адмін-панель не індексується пошуковиками (noindex).
